## Spacing Scale

Existing CSS tokens map to the 8-point scale. All new elements in this phase MUST use existing `--sp-*` tokens only.

| Token | CSS Var | Value (rem) | Equivalent px | Usage |
|-------|---------|-------------|--------------|-------|
| xs | `--sp-xs` | 0.25rem | 4px | Icon gaps, tight inline padding |
| sm | `--sp-sm` | 0.5rem | 8px | Bank piece internal padding, button gaps |
| md | `--sp-md` | 1rem | 16px | Default element spacing, card padding |
| lg | `--sp-lg` | 1.5rem | 24px | Section padding, overlay gaps |
| xl | `--sp-xl` | 2rem | 32px | Layout gaps, overlay padding |
| 2xl | `--sp-2xl` | 3rem | 48px | Major section breaks |

Source: `client/style.css` lines 47–53 (`--sp-*` tokens)

Exceptions:
- **Rotation buttons minimum touch target: 44px × 44px.** This is a touch accessibility requirement — 44px is below `--sp-2xl` (48px) so use explicit `min-width: 44px; min-height: 44px` on `#rotation-controls button`, not a spacing token. This is the sole spacing exception in this phase.
- Grid cell `gap: 3px` and `.grid { padding: 6px }` are preserved from Phase 6 — do not change.
- `.grid { margin: 1rem auto }` is preserved — do not change.

## Typography

All roles map directly to the existing type scale. No new type sizes are introduced this phase.

| Role | Size | CSS Var | Font Family | Weight | Line Height | Usage |
|------|------|---------|-------------|--------|-------------|-------|
| Body | 16px (1rem) | — | Nunito (`--font-body`) | 400 | 1.5 | General text, game notifications |
| Label | 12px (0.75rem) | — | Nunito (`--font-body`) | 700 | 1.4 | Form labels, bank piece labels |
| Heading | ~30px (1.9rem) | — | Fredoka (`--font-display`) | 700 | 1.2 | Section headings (`h2`), rotation button glyphs, portrait overlay message |
| Display | 48px (3rem) | — | Fredoka (`--font-display`) | 700 | 1.1 | Page title (`h1`), portrait overlay icon |

Source: `client/style.css` lines 98–137 (`h1`, `h2`, `h3` rules)

New element specifics for this phase:

- **Rotation button labels** (`↺` / `↻`): font-size `1.9rem`, font-family `--font-display` (Fredoka), weight `700`. Uses the existing Heading size — glyphs are large enough to tap comfortably and visually consistent with section headings.
- **Portrait overlay message** "Bitte Querformat verwenden": font-size `1.9rem`, weight `700`, `--font-display`. Matches the Heading role to establish a clear, prominent orientation instruction.
- **Portrait overlay icon** (↻): font-size `3rem`, weight `700`, `--font-display`. Uses the existing Display size — provides ample visual dominance for the icon.

Type scale summary for this phase: **4 sizes** (0.75rem, 1rem, 1.9rem, 3rem), **2 weights** (400 regular, 700 bold).

## Color

The existing palette is the only palette. No new color values are introduced.

| Role | CSS Token | Hex | Usage |
|------|-----------|-----|-------|
| Dominant (60%) | `--clr-bg` | #fdf9f3 | Page background, game screen background |
| Secondary (30%) | `--clr-surface` / `--clr-surface-alt` | #ffffff / #f2ece0 | Cards, piece bank panel, rotation controls container |
| Accent (10%) | `--clr-primary` | #ff5533 | Reserved for: active/selected bank piece border, rotation button hover/active state, win card heading |
| Destructive | `--clr-error` | #e03232 | Ghost preview invalid state (`ghost-invalid`), error messages only |

Source: `client/style.css` lines 13–35 (palette + semantic tokens)

**Accent reserved for (explicit list):**
1. Selected bank piece border (`border-color: var(--clr-primary)` on `.bank-piece.selected`)
2. Rotation button hover background: `var(--clr-primary)` with white text — mirrors the default `button` style
3. Rotation button active state: `translate(2px, 2px)` + `box-shadow: none` — same pressed-button illusion as all other buttons in the system
4. The primary CTA button (existing pattern — no change)

**Rotation button default (non-hover) state:** Background `--clr-surface-alt` (#f2ece0), border `2px solid --clr-border`, box-shadow `--sh-sm`, color `--clr-text`. This matches `.bank-piece` visual language — buttons live next to the bank panel and should feel like siblings.

**Portrait overlay background:** `var(--clr-bg)` (#fdf9f3) at full opacity — opaque, not semi-transparent. Ensures unambiguous "device orientation required" signal.

Dark mode: The `.portrait-overlay` and `#rotation-controls` use standard CSS tokens, so dark mode is inherited automatically via `body.dark-mode` overrides in `style.css` lines 659–682. Grid and bank panel remain in original light colors per existing convention.

## Component Inventory

New components introduced in this phase. Executor must implement exactly these.

### 1. `#rotation-controls` — Rotation Button Container

```html
<div id="rotation-controls" class="rotation-controls">
  <button id="rotate-ccw-btn" aria-label="Drehen gegen Uhrzeigersinn">↺</button>
  <button id="rotate-cw-btn" aria-label="Drehen im Uhrzeigersinn">↻</button>
</div>
```

**Placement:** Inside `.game-area` div in `index.html`, adjacent to `#piece-bank`.

**CSS contract:**

```css
.rotation-controls {
  display: flex;
  gap: var(--sp-sm);           /* 8px between the two buttons */
  justify-content: center;
  margin-top: var(--sp-sm);    /* 8px above, below bank panel */
}

.rotation-controls button {
  width: auto;
  min-width: 44px;             /* touch target — accessibility exception */
  min-height: 44px;            /* touch target — accessibility exception */
  padding: var(--sp-xs) var(--sp-sm);  /* 4px 8px */
  font-family: var(--font-display);
  font-size: 1.9rem;
  font-weight: 700;
  background: var(--clr-surface-alt);
  color: var(--clr-text);
  border: 2px solid var(--clr-border);
  border-radius: var(--r-md);
  box-shadow: var(--sh-sm);
  cursor: pointer;
  transition: transform var(--t-fast), box-shadow var(--t-fast), background var(--t-fast);
}

.rotation-controls button:hover:not(:disabled) {
  background: var(--clr-primary);
  color: #fff;
  transform: translate(-1px, -1px);
  box-shadow: var(--sh-md);
}

.rotation-controls button:active:not(:disabled) {
  transform: translate(2px, 2px);
  box-shadow: none;
}

.rotation-controls button:disabled {
  background: var(--clr-surface-alt);
  color: var(--clr-text-faint);
  box-shadow: var(--sh-xs);
  cursor: not-allowed;
  opacity: 0.5;
}
```

**Disabled state:** Both buttons are disabled when no piece is selected (`selectedShapeId === null`). Disable via `button.disabled = !selectedShapeId` after each state change.

**Deselect exclusion:** The `#rotation-controls` div MUST be added to the document-level deselect exclusion check in `main.js`. See RESEARCH.md Pitfall 3.

### 2. `.portrait-overlay` — Portrait Mode Overlay

```html
<div class="portrait-overlay" role="alert" aria-live="polite">
  <span class="portrait-overlay__icon" aria-hidden="true">&#8635;</span>
  <p class="portrait-overlay__message">Bitte Querformat verwenden</p>
</div>
```

**Placement:** As the last child of `<body>` in `index.html`, before `</body>`.

Note: The `&#8635;` is the rotation arrow symbol (U+21BB ↻). Do not use an emoji — emojis render inconsistently across tablet browsers. If preferred, replace with the text "↻" directly.

**CSS contract:**

```css
.portrait-overlay {
  display: none;                /* hidden by default */
  position: fixed;
  inset: 0;
  z-index: 9999;
  background: var(--clr-bg);   /* #fdf9f3 — fully opaque */
  align-items: center;
  justify-content: center;
  flex-direction: column;
  gap: var(--sp-lg);            /* 24px between icon and message */
  font-family: var(--font-display);
  font-size: 1.9rem;
  font-weight: 700;
  text-align: center;
  padding: var(--sp-xl);        /* 32px all sides */
  color: var(--clr-text);
}

@media (orientation: portrait) {
  .portrait-overlay {
    display: flex;
  }
}

.portrait-overlay__icon {
  font-size: 3rem;
  line-height: 1;
  color: var(--clr-primary);   /* red rotation icon — on-brand */
}

.portrait-overlay__message {
  margin: 0;
  max-width: 280px;
}
```

**Implementation approach:** Pure CSS `@media (orientation: portrait)` — no JS resize listener. See RESEARCH.md Pattern 7 rationale.

### 3. `--cell-size` CSS Custom Property — Auto-Scaling Grid

**Not a visible component.** A CSS token change to `style.css` and corresponding JS update.

**CSS contract:**

```css
:root {
  /* Existing tokens remain unchanged. Add: */
  --cell-size: min(calc((100vw - 240px) / 9), 60px);
  /* Formula: 9 columns, ~240px reserved for bank+controls+gap, cap at 60px desktop */
}

.grid-cell {
  /* Replace fixed 40px with: */
  width: var(--cell-size);
  height: var(--cell-size);
  font-size: calc(var(--cell-size) * 0.22);
  /* All other .grid-cell rules unchanged */
}

#game-screen {
  /* Replace max-width: 960px with: */
  max-width: min(calc(100vw - 2rem), 1200px);
  text-align: center;
}
```

**JS contract (renderGrid update):**

```javascript
// Replace hard-coded '40px' in gridTemplateColumns/Rows with:
gameGrid.style.gridTemplateColumns = `repeat(${cols}, var(--cell-size))`;
gameGrid.style.gridTemplateRows    = `repeat(${rows}, var(--cell-size))`;
```

Source: RESEARCH.md Pattern 6 + Pitfall 5

## Interaction Contract

All interaction behaviors are locked by CONTEXT.md decisions. This section provides the visual contract for each state.

### Click / Tap Model (Desktop + Touch)

| Action | Visual Feedback |
|--------|----------------|
| Bank piece selected | `.bank-piece.selected` — red border, `--clr-primary-bg` background, `-2px -2px` translate (existing CSS) |
| Hover over empty grid cell (piece selected) | `.ghost-valid` — green semi-transparent fill + dashed green border (existing CSS) |
| Hover over occupied/invalid cell (piece selected) | `.ghost-invalid` — red semi-transparent fill + dashed red border (existing CSS) |
| Single click on empty cell (piece selected) | Piece placed — cell re-renders as `.placed` with piece color; bank piece removed from bank |
| Single click on placed movable piece (no selection) | Piece returned — cell reverts to `.empty`; piece re-appears in bank |
| Click on inactive cell | No response — `pointer-events: none` prevents event (existing) |

### Touch Drag Model (Tablet)

| Action | Visual Feedback |
|--------|----------------|
| `touchstart` on bank piece | Same as bank piece click — `.bank-piece.selected` state applied |
| `touchmove` over grid | Ghost preview follows finger — same `.ghost-valid` / `.ghost-invalid` cells as desktop hover |
| `touchend` on grid | Ghost stays in place — highlighted cells remain until next tap confirms placement |
| Tap ghost-highlighted cell | Placement confirmed — same visual as single-click place above |
| Finger drags off grid | `clearGhostPreview()` called — ghost cells cleared; piece remains selected in bank |
| Long press (500ms) on placed movable piece | Return to bank — same visual as single-click return above |

### Rotation Buttons

| State | Visual |
|-------|--------|
| No piece selected | Both `#rotate-ccw-btn` and `#rotate-cw-btn` are `disabled` — opacity 0.5, `cursor: not-allowed`, `--clr-text-faint` color |
| Piece selected, idle | Default state — `--clr-surface-alt` background, `--clr-border` border, `--sh-sm` shadow |
| Button hovered | `--clr-primary` background, white text, `-1px -1px` translate, `--sh-md` shadow |
| Button pressed | `+2px +2px` translate, `box-shadow: none` — pressed-button illusion (matches all other buttons) |
| After rotation | Ghost preview and bank mini-grid update immediately — no transition delay |

### R Key Shortcut

| State | Visual Feedback |
|-------|----------------|
| R pressed with piece selected | Same as clicking `#rotate-cw-btn` — ghost + bank mini-grid update; no additional visual indicator needed |
| R pressed with no piece selected | No visual response — no piece to rotate |
| R pressed outside game screen | No response — guard via `gameScreen.classList.contains('active')` |

## Copywriting Contract

All player-facing strings are in German, consistent with the existing UI convention (source: CONTEXT.md `## Established Patterns`).

| Element | Copy (German) | Notes |
|---------|---------------|-------|
| Rotate CW button label (aria) | "Drehen im Uhrzeigersinn" | Aria-label only — visual is ↻ symbol |
| Rotate CCW button label (aria) | "Drehen gegen Uhrzeigersinn" | Aria-label only — visual is ↺ symbol |
| Portrait overlay message | "Bitte Querformat verwenden" | Source: CONTEXT.md decision |
| Long-press return (no notification) | (none) | Return is instant with no confirmation — matches desktop single-click return behavior |
| Placement success (no notification) | (none) | Placement is instant visual feedback only — existing behavior |
| Placement rejected (invalid position) | (existing error via `showGameNotification`) | No new copy needed — server already returns rejection |

**Primary CTA for this phase:** There is no new primary CTA screen. The rotation buttons are the primary new interactive elements — their labels are defined above.

**Empty state:** Not applicable for this phase — the bank panel empty state is unchanged from Phase 6.

**Error states (touch-specific):**
- Ghost on invalid cell: `.ghost-invalid` visual (red) — no text needed. Red ghost is sufficient signal.
- Touch off grid (no ghost): No notification. Piece remains selected. Silence is intentional.
- Long-press on anchor cell: No response. Anchor pieces are not movable — no notification needed.

**Destructive actions in this phase:** None. "Return piece to bank" is non-destructive (piece goes back to bank, not deleted). No confirmation dialog needed.
