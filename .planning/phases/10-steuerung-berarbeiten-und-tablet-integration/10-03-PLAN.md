Present the following verification checklist to the user. Start the server (`cd server && node src/server.js`) and open http://localhost:3000.

**Desktop verification (browser):**
1. Create room, join with second tab, start game
2. Select a bank piece — rotation buttons (CW/CCW) should become active (not grayed)
3. Click CW rotation button — bank mini-grid rotates 90 degrees CW; ghost updates if hovering grid
4. Click CCW rotation button — bank mini-grid rotates 90 degrees CCW
5. Press R key — piece rotates 90 degrees CW (same as CW button)
6. Click an empty grid cell — piece places INSTANTLY (no 150ms delay)
7. Click rotation button — piece should NOT be deselected after rotation
8. Click outside grid/bank/rotation buttons — piece IS deselected, buttons become disabled
9. With no piece selected, click a placed movable piece — piece returns to bank
10. Resize browser window from wide to narrow — grid cells scale fluidly (not fixed 40px), capped at ~60px on wide screens

**Touch verification (Chrome DevTools > Toggle Device Toolbar > iPad landscape 1024x768):**
11. Touch a bank piece — it becomes selected
12. Drag finger from bank to grid — ghost preview follows finger in real-time
13. Lift finger from grid — ghost stays highlighted at last position
14. Tap the ghost-highlighted cell — piece is placed
15. Select piece, drag to grid, drag finger OFF the grid entirely — ghost clears
16. Place a piece, deselect, long-press (~500ms hold) on placed piece — piece returns to bank
17. Short-tap placed piece (less than 500ms) with no piece selected — piece returns to bank (normal click)

**Portrait overlay:**
18. In DevTools, switch to iPad portrait (768x1024) — "Bitte Querformat verwenden" overlay should appear with rotation icon
19. Switch back to landscape — overlay disappears

**Regression checks:**
20. Play a full game through to win — win overlay shows correct time
21. Ghost preview on invalid positions shows red dashed border
22. Anchor pieces cannot be clicked/returned
23. Inactive corner cells have no visual or interaction response

Wait for user to type "approved" or describe issues.
