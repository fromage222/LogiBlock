    3. Order of operations inside the click handler when `pendingRotate === true` and `selectedShapeId !== null` (the existing single-use guard from 14-02):
       a. `selectedRotation = (selectedRotation + 90) % 360;`

       d. `showGameNotification('Piece rotated!')` or whatever the existing trap notification text is — keep the exact same string that the old setTimeout version used.
    4. Do NOT add any delay, requestAnimationFrame, or async wrapper. The rotation must be visible on the very next paint, before any grid click can emit game:move.
