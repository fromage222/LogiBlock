  <name>Task 1: Human verification — single-click rotate, double-click place/return, ghost sync, bank select-only</name>
  <files>none</files>
  <action>Human verifies all Phase 7 interaction model behaviors: single-click rotates (with 300ms disambiguation), double-click places/returns, ghost preview and bank mini-grid immediately reflect rotation, bank click selects/deselects only.</action>
  <what-built>
    Complete Phase 7 new interaction model in client/main.js:
    - Single-click on active grid cell: rotates selected piece 90° CW (300ms disambiguator prevents rotate on double-click)
    - Double-click on active grid cell: places selected piece (no extra rotation), or returns placed piece if none selected
    - Ghost preview redraws immediately after rotation without requiring mouse movement
    - Bank mini-grid for selected piece redraws immediately after rotation
    - Bank click: select-or-deselect only (always resets rotation to 0°), no longer rotates on second click
  </what-built>
  <how-to-verify>
    Start the server first:
    ```
    node server/src/server.js
    ```
    Open the game in a browser (http://localhost:3000), create a room, start a game with the Corner Cut puzzle.

    **Test 1 — Single-click rotates:**
    1. Click any bank piece to select it
    2. Single-click any active (non-transparent) grid cell
    3. Wait ~350ms (past the disambiguation window)
    Expected: The piece in the bank mini-grid has rotated 90° CW. The ghost preview (if hovering) shows the new orientation. The piece is still selected.

    **Test 2 — Multiple rotations cycle through 0→90→180→270→0:**
    1. Select a piece from the bank
    2. Single-click a grid cell 4 times (waiting ~350ms between each)
    Expected: After 4 rotations, the piece is back to its original 0° orientation.

    **Test 3 — Double-click places without extra rotation:**
    1. Select a piece, optionally rotate it once via single-click
    2. Note the current rotation in the bank mini-grid
    3. Double-click a valid empty cell where the piece fits at that rotation
    Expected: The piece is placed at exactly the rotation shown in the bank (no extra rotation from the double-click). The piece disappears from the bank.

    **Test 4 — Ghost updates after rotation without mouse movement:**
    1. Select a piece, hover over a grid cell (ghost appears)
    2. Without moving the mouse, single-click the cell
    3. Wait ~350ms
    Expected: The ghost preview on that cell updates to show the rotated piece orientation. You did NOT move the mouse — the ghost still updated.

    **Test 5 — Bank click resets to 0°, second click deselects:**
    1. Select a piece and rotate it (via grid single-click) to 90° or 180°
    2. Click the same piece in the bank
    Expected: Piece is deselected (no longer highlighted).
    3. Click the piece again
    Expected: Piece is re-selected at 0° (NOT at the previous rotation). Bank mini-grid shows 0° orientation.

    **Test 6 — Return piece via double-click:**
    1. Place a piece on the grid (via double-click)
    2. Deselect all pieces (click outside grid/bank)
    3. Double-click the placed piece on the grid
    Expected: The piece returns to the bank.

    **Test 7 — Single-click with no piece selected does nothing:**
    1. Deselect all pieces
    2. Single-click any empty active grid cell
    3. Wait ~350ms
    Expected: Nothing happens. No error. No state change.
  </how-to-verify>
  <resume-signal>Type "approved" if all 7 tests pass, or describe which tests failed with observed vs. expected behavior.</resume-signal>
